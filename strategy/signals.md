# Signal Framework

## Purpose

Signals convert hard data and market behavior into an operating posture. They do not replace the structural thesis. A great company can still be a bad entry if the signal state is poor.

## Inputs

Use four inputs:

| Input | What to check |
| --- | --- |
| Hard data | Revenue, backlog, margins, capex, capacity, users, contracts, product adoption |
| Market emotion | Volatility, trend, sentiment, positioning, crowding |
| Valuation | Multiple, growth expectations, free cash flow, unit economics, comparison to history |
| Structural evidence | Layer importance, moat, cost curve, regulatory support, ecosystem role |

## Traffic Lights

| Signal | Condition | Default action |
| --- | --- | --- |
| Green | Hard data healthy, thesis improving, market emotion normal or supportive | Add in stages or hold full target |
| Yellow | Data mixed, valuation fair to rich, market starting to diverge | Hold, add slowly, or wait for confirmation |
| Orange | Data weakening or sentiment overheated, downside risk rising | Pause adds, trim tactical exposure, tighten review |
| Red | Data materially deteriorating, thesis damaged, or market panic/mania dominates | Exit, hedge, or move to watchlist |

## Signal-To-Action Map

| Current state | Existing holding | New idea |
| --- | --- | --- |
| Green | Hold or add toward target | Start position if valuation is acceptable |
| Yellow | Hold, add only on confirmation | Watchlist or starter only |
| Orange | Trim T3/T4 first, stop adding | Do not start unless crisis plan exists |
| Red | Exit or reduce to thesis-monitor size | Avoid |

## Tactical Overlay Rules

- Use charts for timing, not for thesis creation.
- Do not average down automatically. Add only when the signal improves or valuation compensates for the risk.
- A green signal allows adding, but does not force buying if valuation is excessive.
- A red signal can force exit even if the long-term story remains appealing.
- When signal state conflicts with structural thesis, reduce position size until the conflict resolves.

## Event Triggers

Run an event review when any of these occur:

- Earnings or guidance change.
- Product or platform milestone.
- Major customer win or loss.
- Regulation, export control, antitrust action, or subsidy change.
- Supplier bottleneck or capacity expansion.
- 20% or greater price move without clear fundamental news.
- Broad market shock affecting AI valuations.

## Required Output For Each Signal Review

Every signal review should end with one of these actions:

- Add.
- Hold.
- Trim.
- Pause.
- Exit.
- Watchlist.

The output must include a short reason, the affected L layer, the affected T tier, and the next trigger that would change the action.
